Comment lier une branche distante à un dépôt local

Si en théorie Git impose une séparation claire entre les branches locales et distantes, en pratique il peut être avantageux de les lier, les considérant ainsi comme des copies toujours fidèles l'une de l'autre.

Pourquoi lier une branche distante à une branche locale ?

Imaginons que vous travaillez sur une branche locale nommée "developpement" et qu'elle soit liée à une branche distante "origin/developpement". Cette relation va s'avérer très utile pour deux raisons :

  1. les commandes pull et push deviennent beaucoup plus simple. Avec une branche liée, vous pouvez désormais utiliser ces commandes dans leur mode raccourci git pull et git push sans avoir à vous soucier des paramètres exacts - dans cet exemple git push origin developpement.
  2. En outre Git va fournir un suivi des fichiers que vous n'avez pas encore synchronisé avec push et pull. Voici un exemple :
  1. si vous avez 2 commits dans votre dépôt local que vous n'avez pas encore envoyé sur le distant, alors git vous signalera que votre branche locale se trouve "en avance de 2 commits" sur la distante.
  2. réciproquement, si 4 commits se trouvent sur la branche distante et que vous n'avez fait aucun pull, alors git vous précisera que votre branche locale est "en retard de 4 commits" par rapport à la branche distante.
